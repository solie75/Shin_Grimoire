### [Create An Amazon GameLift Anywhere fleet](https://docs.aws.amazon.com/gameliftservers/latest/developerguide/fleets-creating-anywhere.html)

Amazon GameLift Anywhere 을 aws 서버가 아닌 자체 장비를 GameLift 시스템에 연결하여 사용하는 기능이다. 자체 서버를 사용하지만 GameLift 세션관리 기능(매치 메이킹, 세션 매치 등) 은 그대로 이용할 수 있다.

- 다음 3단계 준비
1. Server SDK 5.x 버전 이상 사용.
2. 게임 서버 실행 파일 + DLL 등 필요한 라이브러리 + 설정 파일 등 서버를 돌리는데 필요한 모든 파일을 하나로 모아 패키지로 만들어야 한다.
3. Agent 사용 여부 결정 :  GameLift Agent 란 자체 서버 컴퓨팅 자원에 설치되어, 게임 프로세스를 켰다 껐다 관리하고 상태를 보고하는 자동화 도구이다.

![AWS_GameLiftConsole_AnywhereFleet_Details](/Image/Server/AWS_GameLiftConsole_AnywhereFleet_Details.png)
- 생성된 fleet 은 고유한 fleet id 와 ARN 을 가지며 현재 Active 상태 인 것을 확인할 수 있다.
- [GameLift 콘솔에서 Fleet 의 동작 상태를 두 가지 탭으로 확인할 수 있다.](https://docs.aws.amazon.com/gameliftservers/latest/developerguide/monitoring-gamelift-servers-metrics.html)
	- Metrics(지표) : 데이터가 수집되는 즉시 활성 인스턴스(게임 서버) 수, 게임 세션 및 플레이어 세션의 상태( 활성, 대기 중 등) 를 그래프로 보여준다.
		- 지표 종류
			- Active instances – The fleet's total hosting capacity. Includes instances that are hosting or are available to host game sessions.
			- Idle instances – Instances that are not hosting any game sessions.
			- Percent idle instances – Instances that are not hosting game sessions, expressed as a percentage of total active instances.
			- Desired instances – The target number of active instances.
			- Max instances – The configured maximum capacity for the fleet.
			- Min instances – The configured minimum capacity for the fleet.
			- Recycled instances – Spot instances that have been recycled and replaced due to likelihood of interruption.
			- Instance interruptions – Spot instance interruptions.
			- Activating game sessions – Game sessions that are in the process of starting up.
			- Active game sessions – The fleet's total number of game sessions in progress.
			- Available game sessions – Server processes that are ready to host a new game session. There may be a delay due to limits on game session activation.
			- Percent available game sessions – Server processes that are ready to host a new game session without delay, expressed as a percentage of total server processes.
			- Concurrent activatible game sessions – Server processes that are ready to host a new game session without delay.
			- Game session interruptions – Game sessions on Spot instances that have been interrupted.
			- Current player sessions – Players who are connected to a game session or have a reserved slot in a game session.
			- Player session activations – Players who connected to a game session and claimed their reserved slot.
	- Events(이벤트) : 플릿 생성, 위치 연결 등 플릿 생명주기와 관련된 모든 사건을 타임스탬프와 함께 기록한다.

##### [Game Session 생성.](https://docs.aws.amazon.com/cli/latest/reference/gamelift/create-game-session.html)

Game Session 은 플레이어가 참여할 때 이를 추적하는 AWS 호스팅 리소스이다.

서버를 키고 cmd 에 다음 명령어를 입력한다.

aws gamelift create-game-session --fleet-id fleet-d71c98fe-d191-4059-9b58-eaf53be37e37 --name DebugGameSession --maximum-player-session-count 4 --region ap-northeast-2 --location custom-home-location --profile shin

게임 세션의 이름을 DebugGameSession 이라 짓고 최대 플레이어 세션 수를 4 로 제한한다.


콘솔의  Fleet $\to$ GameSession 항목을 보면 새로 추가된 것을 확인 할 수 있다.

또한 Dedicated Server Iog 창을 보면 다음과 같은 문구가 있다.

```cmd
[2025.12.17-09.11.48:651][485]LogShooterGameMode: GameSession Initializing : arn:aws:gamelift:ap-northeast-2::gamesession/fleet-d71c98fe-d191-4059-9b58-eaf53be37e37/custom-home-location/gsess-f4d4b278-5cae-4b07-b6ca-11ce15c0ddfb
```

이는 이전에 c++ 코드 상에서 미리 UE_LOG 로 지정했기 때문이다.

```c++
auto OnGameSession = [=](Aws::GameLift::Server::Model::GameSession gameSession)  
{  
    FString GameSessionId = FString(gameSession.GetGameSessionId());  
    UE_LOG(LogShooterGameMode, Log, TEXT("GameSession Initializing : %s"), *GameSessionId);  
    GameLiftServerSDKModule->ActivateGameSession();  
};
```

해당 람다 코드를 통해서 게임 세션이 ActivateGameSession 함수를 호출한다. 

정리하자면 
CLI 로 'create-game-session' 입력 $\to$  GameLift 가 빈 서버를 찾아서 서버에게 방을 만들라는 신호를 보냄 $\to$  Server Code 는 OnGmaeSession 람다 함수를 발동하고 해당 함수 내의 ActivateGameSession 이 호출됨 $\to$ AWS Console 에서 GameSession 의 상태가 Active 로 변경됨.

##### [Player Session 생성](https://docs.aws.amazon.com/cli/latest/reference/gamelift/create-player-session.html)

다음 명령어 입력
```cmd
C:\Users\solie>aws gamelift create-player-session --game-session-id arn:aws:gamelift:ap-northeast-2::gamesession/fleet-d71c98fe-d191-4059-9b58-eaf53be37e37/custom-home-location/gsess-23b244d1-e3cf-444f-9ed9-85c7337d6c55 --player-id PlayerOne --profile shin
```

다음과 같이 플레이어 세션이 생성된다.
```cmd
{
    "PlayerSession": {
        "PlayerSessionId": "psess-a72455b2-84d3-4f7a-81ac-c0fc1730aa24",

        "FleetId": "fleet-d71c98fe-d191-4059-9b58-eaf53be37e37",
```

aws console 의 Game Session 의 가로 Details 의 Player Session 에 보면 
![[AWS_PlayerSession_In_Console.png]]
해당 상태는 플레이어가 Game Session 을 배정받은 상태로 Reserved 상태이다. 이때 일정 시간이 지나도록 서버에 접속하지 않는다면 Timeout 상태가 되기 때문에 그 시간 내에 AcceptPlayerSession 함수를 호출해서 Active 상태가 되어야 한다.

플레이어 세션은 친구 따라가기나 파티 플레이 등의 기능에 활용된다.
따라가고 싶은 특정 친구를 검색하기 위해서는 해당 친구가 있는 서버의 IP 주소를 알아야 하는데 이는 GameLift 에 있는 플레이어 세션만 조회하면 되기 때문이다.
