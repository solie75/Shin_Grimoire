Hypertext Transfer Protocol 의 약자.

웹상에서 이루어지는 모든 데이터 교환의 기초. 서버와 브라우저 간에 하이퍼텍스트 요청과 정보를 전송하는 데 사용되는 프로토콜이다.

HyperText 란 컴퓨터 화면이나 기타 장치에 표시되는 텍스트로 독자가 접근할 수 있는 다른 텍스트로의 참조나 하이퍼 링트를 포함한다.

HTTP 요청은 정보를 요청하거나 어떤 작업을 수행해 달라고 서버에게 의사를 보내는 단방향 통신을 말한다.

- HTTP 구성
	- Method
		- 어떤 유형의 작업을 요청하는지를 나타낸다.
		- get, put, post, delete 등
	- URL
		- 접근하려는 리소스의 주소
	- Header
		- 콘텐츠 유형, 인증 등 요청에 대한 메타데이터를 포함한다.
	- Body
		- 요청과 함께 보내는 데이터가 담긴 것으로 선택적 옵션이다.
		- 주로 put 이나 Post 메서드와 함께 사용된다.
- HTTP Method
	- Get
		- 서버에서 데이터를 가져올 때 사용한다. (ex. 사용자의 프로필 정보 가져오기 등)
		- 해당 작업은 안전하고 멱등(idempotent) 하다.
	- Post
		- 서버에 새로운 리소스를 생성할 때 사용한다. (ex. 새로운 사용자 계정을 만들기 위해 가입 양식을 제출하는 것 등)
		- 해당 작업은 멱등하지 않다. 즉, 같은 요청을 반복하면 리소스가 여러 개 생성된다는 뜻.
	- Put
		- 기존 리소스를 업데이트 하거나, 리소스가 없다면 새로 만들 때 사용된다. (ex. 사용자의 세부 정보나 설정을 업데이트 하는 것 등)
		- 해당 작업은 멱등하다. 같은 요청을 반복해도 결과가 같기 때문.
	- Delete
		- 서버에서 리소스를 삭제할 때 사용된다. (사용자의 계정을 삭제하는 것 등)
		- 해당 작업은 멱등하다.
