네트워크에서 Port 란 운영체제 내에서 실행 중인 특정 프로세스(프로그램) 를 식별하기 위한 16비트의 논리적 식별자(번호) 이다.
- IP 주소 : 네트워크 상에서 특정 컴퓨터(호스트) 를 찾는 주소
- Port 번호 : 그 컴퓨터 안에서 데이터를 받을 구체적인 프로세스(애플리케이션) 를 구분하는 번호.
하나의 IP 주소(서버 컴퓨터) 에는 여러 프로그램( 게임서버, 웹 서버, SSH 등) 이 동시에 실행될 수 있다. 외부에서 패킷(데이터) 이 도착했을 때, 이 패킷이 어떤 프로그램으로 전달되어야 하는지를 결정하는 것이 포트 번호이다.

- 게임 서버에서의 동작 원리
	- Binding : 게임 서버 프로새스가 실행될 때, 운영체제 에게 '나는 UDP 7777 번 포트를 점유하겠다' 라고 요청한다. 이를 소켓 바인딩 이라 한다.
	- Listening : 바인딩이 성공하면, 서버는 해당 포트(7777)로 들어오는 모든 트래픽을 감시하고 받아들일 준비를 한다.
	- Multiplexing (다중 프로세스) : 하나의 서버 컴퓨터 (VM) 에서 여러 개의 게임 세션을 동시에 돌리고 싶다면, 각 프로세스는 서로 다른 포트(7777, 7778, 7779 등)을 가져야 한다. 포트가 겹치면 충돌(Port Conflict) 가 발생하여 서버가 실행되지 않는다.
- GameLift SDK 코드의 의미
	- FProcessParameters.port 변수를 설정하는 행위는 GameLift 서비스에 메타데이터를 등록하는 과정이다.
		- "현재 실행된 이 게임 서버 프로세스는 OS 상에서 7777 번 포트에 바인딩 되어 있다." 라고 GameLift 백엔드에 값을 전달한다.
		- GameLift 는 이 정보를 저장해 두었다가, 클라이언트(플레이어) 가 매칭을 요청하면 (서버의 공격 IP) : (7777) 이라는 접속 정보를 생성하여 클라이언트에게 반환한다.
	- 즉 포트는 서버 컴퓨터 내에서 실행 중인 '게임 서버 프로세스' 의 고유한 네트워크 주소 번호이며, 코드는 이 번호를 GameLift 시스템에 알리는 (Report) 역할을 한다.
