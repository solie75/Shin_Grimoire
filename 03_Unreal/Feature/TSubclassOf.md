특정 클래스와 그 자식 클래스들의 정보(UClass) 만 담을 수 있는 변수 타입.

- 블루프린트
	- c++ 클래스와 블루프린트 에셋 간의 참조(Reference) 방식과 타입 안전성 때문.
	1. Decoupling : 하드 코딩 방지
		- c++ 코드 내에서 특정 블루프린트 파일을 직접 로드하려면 파일 경로를 문자열로 적어야 한다. 
		- 이렇게 하면 파일 위치를 옮기거나 이름을 바꿀 때마다. 파일경로 문자열을 업데이트 해주어야 한다.
		- TSubclassOf 변수를 UPROPERTY(EditDefaultsOnly) 로 선언하면 C++ 코드는 비워두고 Editor 의 디테일 패널에서 블루프린트 에셋을 할당할 수 있다.
- 타입 안정성 보장
	- 단순이 UClass* 타입을 사용하면, 에디터에서 선택할 때 엉뚱한 클래스를 넣을 위험이 있다.
	- TSubclassOf<~> 라고 명시하면, 에디터는 ~를 상속 받은 클래스만 목록에 필터링하여 보여준다. 컴파일 타임과 데이터 레벨에서 오류를 방지한다.
		-  인스턴스 포인터 (AWeapon* MyWeapon)
			- 실제로 게임세계에 존재하는 무기 객체 하나를 가리킨다.
			- 실제 행동을 시킬 때 사용.
			- 아직 생성되지 않은 무기는 가리킬 수 없다.
		- 일반 클래스 포인터 (UClass* WeaponType)
			- 무기든, 캐릭터든, 사운드든, 클래스 정보(설계도)라면 무엇이든 담을 수 있다.
			- 광범위 하기 때문에 다른 클래스 정보를 넣어도 에러가 너지 않는다. 나중에 해당 포인터를 사용하려고 할때 크래쉬가 날 수 있다.
		- 안전한 클래스 포인터 (TSubclassOf\<AWeapon> WeaponType;)
			- 클래스 정보(설계도)를 담는데, 오직 AWeapon 이거나 그 자식 클래스의 설계도만 담을 수 있다
				- 다른 엉뚱한 클래스가 담기는 것을 코드 차원에서 방지한다.
				- 언리얼 에디터 상세 패널에서 드롭다운을 열면 관련된 블루프린트만 필터링하여 보여준다.
